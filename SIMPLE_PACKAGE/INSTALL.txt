========================================
INSTALLATION STEPS
========================================

1. UPLOAD FILES
   - Upload all files to your cPanel public_html directory
   - Ensure file permissions are set correctly:
     * Folders: 755
     * Files: 644
     * api/ folder: 755

2. CREATE DATABASE
   - In cPanel, go to MySQL Databases
   - Create a new database: mbeki_healthcare
   - Create a database user and assign to the database
   - Grant ALL PRIVILEGES to the user

3. IMPORT DATABASE SCHEMA
   - In cPanel phpMyAdmin, select your database
   - Click "Import" tab
   - Upload and execute the database.sql file

4. CONFIGURE ENVIRONMENT
   - Rename .env.example to .env
   - Edit .env with your database credentials:
     
     DB_HOST=localhost
     DB_NAME=your_database_name
     DB_USER=your_database_user
     DB_PASS=your_database_password

5. TEST INSTALLATION
   - Visit your domain: https://yourdomain.com
   - You should see the login dashboard
   - Check "Settings" tab to verify database connection

========================================
SECURITY SETUP
========================================

1. SECURE THE API FOLDER
   Add this to api/.htaccess:

   <Files "*.php">
       Order Allow,Deny
       Allow from all
   </Files>
   
   <Files "config.php">
       Order Allow,Deny
       Deny from all
   </Files>

2. SSL CERTIFICATE
   - Enable SSL/HTTPS in cPanel
   - Force HTTPS redirects

3. REGULAR BACKUPS
   - Set up automatic database backups
   - Use the built-in export feature regularly

========================================
TROUBLESHOOTING
========================================

DATABASE CONNECTION ISSUES:
- Check database credentials in .env file
- Verify database user has correct privileges
- Ensure database exists and is accessible

PHP ERRORS:
- Check PHP error logs in cPanel
- Ensure PHP version is 7.4+
- Verify all required PHP extensions are installed

PERMISSION ERRORS:
- Set correct file permissions (folders: 755, files: 644)
- Ensure web server can read/write files

FILE UPLOAD ISSUES:
- Check PHP upload_max_filesize setting
- Verify post_max_size is adequate
- Ensure tmp directory is writable
